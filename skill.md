# 技术人员学术尽调评估 Skill

**依赖要求**：使用本 Skill 前必须先安装 agent-browser
```bash
npx skills add https://github.com/vercel-labs/agent-browser --skill agent-browser
```

## ⚠️ 强制要求：必须使用浏览器自动化工具

**所有数据必须通过浏览器自动化工具（如 agent-browser）实时爬取，禁止使用 WebSearch 或其他方式估算。**

支持的浏览器自动化工具：
- **agent-browser**（推荐）: `npx skills add https://github.com/vercel-labs/agent-browser --skill agent-browser`
- **browser-use**: `pip install browser-use` 或其他浏览器自动化方案
- **Claude 桌面端内置浏览器**: 通过截图和交互获取数据

### Google Scholar 数据采集
必须先用浏览器自动化工具打开目标人物的 Google Scholar 主页，获取：
1. 总引用量、h-index、i10-index
2. 按引用排序的论文列表
3. 每篇论文的作者位置（一作/二作）、会议/期刊、引用数

### GitHub 数据采集
必须使用浏览器自动化工具访问 GitHub 页面，获取：
1. 个人主页：Followers、贡献记录
2. 项目页面：Stars、Forks、 Contributors 排名
3. 同类项目对比数据

## 数据采集方式

### Google Scholar 数据
```
1. 打开 https://scholar.google.com/citations?user=[USER_ID]&sortby=citstdirection
2. 获取：总引用量、h指数、i10指数
3. 获取按引用排序的论文列表
4. 记录每篇论文的：作者位置（一作/二作/其他）、会议/期刊、引用数、年份
5. 统计一作/二作顶会顶刊引用量 = 一作顶刊顶会引用量 + 二作顶刊顶会引用量（两者都要求是顶刊顶会）
```

### 个人背景验证（必须执行）
```
1. 搜索 "[姓名] + [机构] + biography/education/CV"
2. 使用浏览器自动化工具打开 researcher 的个人主页/实验室网站
3. 验证教育背景：本科、硕士、博士院校和专业
4. 验证工作经历：当前职位、导师、研究组
5. 查找重要标签（如有）：业内公认的title，例如杨植麟是 Transformer-XL 第一作者
6. ⚠️ 禁止仅根据论文年份推测时间线，必须通过个人网站或可靠来源验证
```

### GitHub 数据
```
1. 搜索 "[姓名] GitHub"
2. 进入正确的 GitHub 主页
3. 获取：Followers、年贡献数
4. 进入各主要项目，获取 Stars、Forks
5. 进入 Contributors 页面，获取 commits 排名
6. 找同类开源项目做 Stars 对比（如：评估姿态估计项目，需对比 OpenPose、MediaPipe 等）
```

### 同赛道对标数据（重点）
```
⚠️ 对标人物选择原则：
1. 优先选择同赛道的知名华人研究员
2. 选择知名具身智能/AI公司背后的创始研究员
3. 避免选择不相关的领域专家

大模型赛道对标人物示例（华人）：
- 杨植麟
- 闫俊杰


获取方式：
1. 搜索 "[研究方向] top Chinese researchers"
2. 搜索 "[研究方向] 创业公司 创始人 学术 华人"
3. 用 agent-browser 获取对标对象的 Google Scholar 数据
```

### 专利数据（如有）
```
1. 从 Google Scholar 论文列表中识别专利（通常标注 "US Patent"）
2. 记录专利引用量
3. 建立论文→专利对应关系
```

## 论文领域梳理排序原则（重要）

### 排序逻辑

**第一步：按研究领域分组**（而非简单按引用量排序）
- 核心研究方向①
- 核心研究方向②
- 新兴方向
- 延伸方向

**第二步：组内综合排序**

| 因素 | 权重 | 说明 |
|------|------|------|
| **一作身份** | 40% | 一作代表主要贡献，应优先展示 |
| **引用量** | 30% | 反映学术影响力 |
| **顶会顶刊** | 20% | CVPR/ICCV/ICRA/TRO/NeurIPS等 |
| **时效性** | 10% | 近期高引论文可能影响更大 |

**第三步：标签标注规则**

| 标签类型 | 格式 | 含义 |
|----------|------|------|
| **核心标签**（加粗） | **精简关键词**（2-4字） | 一作顶会高引论文 |
| 普通标签 | 精简关键词 | 二作或合作顶会论文 |

### 标签 vs 研究内容

| 字段 | 详细程度 | 示例 |
|------|----------|------|
| **标签** | 精简（2-4字） | **抓取基准**、**通用抓取**、**多人姿态** |
| **研究内容** | 详细（1-3句） | 构建了当时最大规模的通用物体抓取数据集，包含97,280张RGB-D图像、88个物体、190个场景，提供超过10亿个抓取标注，成为机器人抓取领域的标准基准 |

## 关键检查清单

在输出报告前，**必须逐项检查**以下要素：

### 结构检查
- [ ] 有"团队成员基本信息"章节（含时间线）
- [ ] 章节标题是问句格式（"学术能力如何？"、"研究方向主要是什么？"）
- [ ] 有"总结来说"开头的总结句
- [ ] 有"数据口径说明"
- [ ] **⚠️ 论文表格必须是一张合并的总表**：不同研究领域用 `| **[领域名]** |||||` 合并单元格分隔，**绝对禁止**拆分成多个独立的子表（如 `#### [领域名]` 下面单独一个表格）
- [ ] 能力总结按重要性排序（1、2、3），区分先后
- [ ] 有"附件"章节

### 内容检查
- [ ] 对标人物选择**同赛道知名华人研究员**或**知名公司创始研究员**
